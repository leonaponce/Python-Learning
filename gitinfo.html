You want to get your changes into the main project so you're not tracking divergent lines of development. There is no absolute upstream repo or downstream repo. Those notions are always relative between two repos and depends on the way data flows: If "yourRepo" has declared "otherRepo" as a remote one, then: -you are pulling from upstream "otherRepo" ("otherRepo" is "upstream from you", and you are "downstream for otherRepo"). -you are pushing to upstream ("otherRepo" is still "upstream", where the information now goes back to). *Note the "from" and "for": you are not just "downstream", you are "downstream from/for", hence the relative aspect. Downstream = when one copies (clone, checkout, etc.) from a repository. Information is flowed "downstream" to you. Working Tree = a directory, which includes files and subdirectories, associated with a repository. Here you can add new files, edit existing files, or remove extraneous files. You access the working tree whenever you open files for a project managed as a Git repository.
